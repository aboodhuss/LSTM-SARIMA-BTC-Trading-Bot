# BTC Trading Bot

Live BTC/USDT paper-trading dashboard with a Python FastAPI backend and a React frontend.

The system streams Binance 1-minute candles, engineers market-structure features, runs an LSTM model for BUY, SELL, and HOLD signals, and tracks a paper-trading portfolio. It also includes a network-resilience test comparing the baseline LSTM against an LSTM + SARIMA repair pipeline.

## What It Shows

- Live BTC/USDT candlestick chart
- LSTM trading signal, confidence, and paper-trading portfolio
- Trade activity and closed-trade blotter
- Network condition telemetry, including delay, packet loss, and dropped candles
- LSTM vs LSTM + SARIMA comparison results under simulated network degradation

## Model Setup

| Model path | Role |
| --- | --- |
| LSTM | Main trading decision model |
| LSTM + SARIMA | Network-resilience path where SARIMA repairs missing candles before LSTM inference |

SARIMA does not replace real candles and does not make the trade decision. It only fills missing candle gaps so the LSTM receives a continuous sequence.

## Project Structure

```text

  scripts/                        Optional network test helpers
```

## Run Locally

Open two terminals.

### Backend

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

### Frontend

```powershell
cd frontend
npm install
npm run dev
```

Open:

```text
http://localhost:5173
```

## Hosting Notes

The frontend can be hosted on Vercel.

The backend should be hosted separately on a service that supports a long-running Python process and WebSockets, such as Render, Railway, Fly.io, a VPS, AWS, Azure, or Google Cloud.

The frontend currently expects the backend at:

```text
http://localhost:8000
ws://localhost:8000/ws
```

For a hosted deployment, update the frontend backend URL logic in `frontend/src/App.jsx` to point at the deployed backend.
